# Структура репозиториев

Проект разделен на четыре репозитория:

## 1. Публичный репозиторий API

**Репозиторий**: `github.com/pyw0w/AniApi` (публичный)

**Содержимое**:
- Пакет `shared` - публичные интерфейсы и типы для разработки плагинов
- Документация для разработчиков плагинов
- Примеры плагинов

**Доступ**: Публичный, доступен всем для разработки плагинов

**Использование**:
```go
import "github.com/pyw0w/AniApi/shared"
```

## 2. Приватный репозиторий ядра

**Репозиторий**: `github.com/pyw0w/AniCore` (приватный)

**Содержимое**:
- `internal/` - внутренние пакеты (не экспортируются)
- `core/` - core-модули (приватные)
- `plugins/` - приватные плагины
- `config.json` - конфигурация

**Доступ**: Приватный, только для команды разработки

**Зависимости**:
- Использует `github.com/pyw0w/AniApi` для публичного API
- Использует `AniDocs` как submodule для документации

## 3. Публичный репозиторий Runtime

**Репозиторий**: `github.com/pyw0w/AniRuntime` (публичный)

**Содержимое**:
- `cmd/runtime` - основной runtime (запускатель)
- Документация по установке и использованию
- Примеры конфигурации

**Доступ**: Публичный, доступен всем для использования

**Зависимости**:
- Использует `github.com/pyw0w/AniApi` для публичного API
- Использует `github.com/pyw0w/AniCore` для ядра (через приватный доступ или локально)
- Использует `AniDocs` как submodule для документации

## 4. Публичный репозиторий документации

**Репозиторий**: `github.com/pyw0w/AniDocs` (публичный)

**Содержимое**:
- Вся документация проекта
- Руководства для разработчиков
- Примеры и туториалы
- API документация

**Доступ**: Публичный, доступен всем

**Использование**: Подключается как git submodule в другие репозитории

## Структура зависимостей

```
Публичный API (AniApi)
  └── shared/ (интерфейсы, типы)

Приватное ядро (AniCore)
  ├── internal/
  ├── core/
  ├── plugins/
  └── docs/ (submodule -> AniDocs)

Публичный Runtime (AniRuntime)
  ├── cmd/runtime
  └── docs/ (submodule -> AniDocs)

Публичная документация (AniDocs)
  ├── plugin-development.md
  ├── repository-structure.md
  ├── api-reference.md
  └── examples/

Публичные плагины (отдельные репозитории)
  └── (используют AniApi/shared)
```

## Разделение ответственности

- **AniApi**: Определяет контракты и интерфейсы для плагинов
- **AniCore**: Содержит реализацию ядра, core-модули и приватные плагины
- **AniRuntime**: Содержит запускатель, который объединяет все компоненты
- **AniDocs**: Централизованная документация для всех компонентов

## Использование AniDocs как submodule

### Добавление submodule

```bash
# В репозитории AniCore или AniRuntime
git submodule add https://github.com/pyw0w/AniDocs.git docs
```

### Обновление submodule

```bash
git submodule update --remote docs
```

### Клонирование с submodules

```bash
git clone --recursive https://github.com/pyw0w/AniCore.git
# или если уже клонирован
git submodule update --init --recursive
```
