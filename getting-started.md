# Быстрый старт

Этот документ описывает, как быстро запустить платформу AniCore локально.

## Предварительные требования

- Установленный Go (версия соответствует `go.mod` в репозиториях).
- Доступ к приватному репозиторию **AniCore**.
- Клонированные репозитории в одну директорию `anidev/`:
  - `AniCore` (приватный)
  - `AniApi` (публичный)
  - `AniRuntime` (публичный)
  - `AniDocs` (публичный, этот репозиторий)

Пример структуры:

```text
/home/USER/anidev/
├── AniCore/
├── AniApi/
├── AniRuntime/
└── AniDocs/
```

## Шаг 1. Настройка go.work

Для локальной разработки используется `go.work`, чтобы все репозитории собирались как единое рабочее пространство.

В корне `anidev/` должен лежать `go.work`, который указывает на все модули:

```text
go 1.25.5

use (
    .              // AniCore
    ./AniApi
    ./AniRuntime
)
```

Если `go.work` ещё не создан или не синхронизирован:

```bash
cd /home/USER/anidev
go work sync
```

## Шаг 2. Настройка конфигурации

Runtime и core-модули используют JSON-конфигурацию. В `AniRuntime` есть пример:

```text
AniRuntime/config.example.json
```

Скопируйте его в `config.json` и настройте под свои нужды:

```bash
cd /home/USER/anidev/AniRuntime
cp config.example.json config.json
# отредактируйте config.json
```

Конфигурация обычно содержит:
- список core-модулей для загрузки (`discord`, `telegram`, `backend` и т.д.),
- путь до `.so` файлов core-модулей,
- путь до `.so` плагинов и список включённых плагинов,
- настройки backend (путь к БД и т.п.),
- уровень логирования.

Подробнее см. `configuration.md`.

## Шаг 3. Сборка core-модулей и плагинов

Сборка выполняется из корня `AniCore` через `make` (требуется `Makefile` в AniCore):

```bash
cd /home/USER/anidev

# Сборка всех компонентов
make build

# Или отдельно:
make build-cores     # только core-модули (.so)
make build-plugins   # только плагины (.so)
```

Убедитесь, что включён `CGO_ENABLED=1`, так как core-модули и плагины собираются как `.so`:

```bash
export CGO_ENABLED=1
```

## Шаг 4. Сборка и запуск runtime

Runtime живёт в репозитории `AniRuntime`:

```bash
cd /home/USER/anidev/AniRuntime

# Запуск напрямую
go run ./cmd/runtime -config config.json

# Либо сборка бинаря (если есть make-таргет)
# make build-runtime
```

При запуске runtime:
- читается `config.json`,
- загружаются core-модули из указанных путей,
- подключается БД (если backend включён),
- загружаются плагины и проверяется их совместимость с версией ядра,
- запускается обработка событий.

Остановка runtime происходит по сигналам `SIGINT`/`SIGTERM` (Ctrl+C).

## Шаг 5. Проверка работы

1. Убедитесь, что все `.so` файлы core-модулей и плагинов находятся в директориях, указанных в `config.json`.
2. Проверьте логи runtime — при успешном запуске будет видно загрузку каждого core-модуля и плагина.
3. Для Discord core-модуля — проверьте, что бот онлайн и slash-команды появились (если есть плагины с `CommandProvider`).
