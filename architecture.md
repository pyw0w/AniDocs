# Архитектура платформы AniCore

Платформа AniCore — это модульная система для построения ботов, разделённая на несколько репозиториев и уровней:

- **AniApi** — публичный слой API, который знают плагины и core-модули.
- **AniCore** — приватное ядро, реализующее core-модули, загрузку плагинов, event bus, безопасность и т.п.
- **AniRuntime** — публичный запускатель, который использует AniCore и AniApi, загружает core-модули и плагины, управляет их жизненным циклом.
- **AniDocs** — документация ко всей системе.

## Структура репозиториев

На верхнем уровне все репозитории живут в директории `anidev/`:

```text
anidev/
├── AniCore/          # Приватное ядро
│   ├── core/         # Core-модули (discord, telegram, backend)
│   ├── plugins/      # Приватные плагины
│   ├── modules/      # Расширения (core/runtime/plugin)
│   ├── pkg/          # Публичные пакеты для runtime
│   ├── data/         # Данные (БД и т.п.)
│   └── ...
├── AniApi/           # Публичный API (shared)
│   └── shared/
├── AniRuntime/       # Публичный runtime
│   └── cmd/runtime/
└── AniDocs/          # Документация
```

### AniApi

`AniApi` содержит только публичные интерфейсы в пакете `github.com/pyw0w/AniApi/shared`:

- `Plugin` — базовый интерфейс плагина.
- `CoreAPI` — API, которое runtime предоставляет плагинам (логирование, события, доступ к БД, вызов core-модулей).
- `CommandProvider` — дополнительный интерфейс для плагинов, которые отдают slash-команды Discord.
- Типы событий (`Event`, типы событий) и вспомогательные типы.
- Версионирование API и диапазоны совместимости (`CompatibilityRange`, `APIVersion`).

`AniApi` **не зависит** от AniCore и AniRuntime и может использоваться сторонними плагинами.

### AniCore

Приватный репозиторий, который реализует:

- **Core-модули** (`core/`):
  - например, `discord`, `telegram`, `backend`.
  - получают `RuntimeAPI` от runtime, работают с событиями и внешними сервисами.
- **Плагины** (`plugins/`):
  - приватные плагины, реализующие `shared.Plugin`.
- **Расширения** (`modules/`):
  - `core-extensions/` — надстройки над core-модулями.
  - `runtime-extensions/` — расширения самого runtime.
  - `plugin-extensions/` — вспомогательные утилиты для плагинов.
- **Публичные пакеты** (`pkg/`):
  - `config` — загрузка конфигурации.
  - `coreloader` — загрузка core-модулей (`.so`).
  - `pluginloader` — загрузка плагинов (`.so`) с проверкой совместимости по версии ядра.
  - `eventbus` — шина событий.
  - `database` — работа с БД и выдача `DatabaseAPI` плагинам.
  - `security` — проверка хэшей и безопасная загрузка артефактов.
  - `logger` — единая система логирования.

AniCore импортирует только:
- `github.com/pyw0w/AniApi/shared` — для публичных интерфейсов,
- собственные пакеты `pkg/...`.

### AniRuntime

`AniRuntime` — тонкий публичный бинарь, который использует AniCore:

- Читает конфигурацию (`config.Config` из `AniCore/pkg/config`).
- Создаёт структуру `Runtime`:
  - инициализирует event bus (`eventbus.EventBus`), загрузчики, security-verifier,
  - читает версию ядра из файла `VERSION`,
  - инициализирует БД (если backend включён) через `database.NewDB`.
- Загружает core-модули (`LoadCores`) и запускает их (`StartCores`).
- Загружает плагины (`LoadPlugins`), проверяя безопасность и совместимость версий.
- Собирает команды плагинов, реализующих `shared.CommandProvider`.
- Управляет жизненным циклом через context и SIGINT/SIGTERM.

Ключевые реализации API:

- `CoreAPIImpl` — реализация `shared.CoreAPI` для плагинов:
  - `Log` — логгирование с префиксом плагина через `pkg/logger`.
  - `Emit` — публикация событий в `eventbus`.
  - `CallCore` — вызов команд core-модулей.
  - `Database` — доступ к `shared.DatabaseAPI` для конкретного плагина.
- `RuntimeAPIImpl` — API для core-модулей:
  - логгирование с префиксом core-модуля,
  - доступ к конфигурации core-модуля,
  - доступ к БД и версиям ядра,
  - получение команд от плагинов для регистрации (например, в Discord).

### AniDocs

`AniDocs` описывает всё выше на человеческом языке:

- как устроена архитектура,
- как использовать публичный API из `AniApi/shared`,
- как писать плагины и core-модули,
- как собирать и деплоить систему.

## Зависимости между репозиториями

```text
AniApi (независимый)
  └── discordgo (для типов)

AniCore
  └── AniApi/shared

AniRuntime
  ├── AniApi/shared
  └── AniCore/pkg/...

Плагины
  └── AniApi/shared
```

Главные правила:

- Плагины **никогда** не импортируют `AniCore/pkg`, только `AniApi/shared`.
- AniCore и AniRuntime используют `AniApi/shared` как стабильный контракт с внешним миром.
- Локальная разработка организована через `go.work`, чтобы одновременно работать с локальными клонами всех репозиториев.

Подробности по разработке и коммитам см. в `development.md`.
